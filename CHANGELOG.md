## [0.1.0] - 2026-07-09

First macOS-native release. Complete port of
[AVRDUDESS](https://github.com/ZakKemble/AVRDUDESS) (WinForms / .NET 2.0) to
Avalonia 12 on .NET 10, running natively on Apple Silicon and shelling out to
a system `avrdude` from Homebrew.

### Added

- **Core** — ported non-UI logic under `src/AvrdudeUI/Core/`:
  `Avrdude` (conf parser + process launcher), `AvrSize`, `CmdLine` (decoupled
  from the UI via a new `AvrdudeSettings` DTO), `Config`, `Presets`,
  `Language`, `MCU` / `Programmer` / `Part`, `MemTypeFile`, `FusesList`,
  `Portable`, `UpdateCheck` (rewritten on `HttpClient`), `XmlFile`.
- **Views** — Avalonia rewrites of every AVRDUDESS form: `MainWindow`,
  `AboutWindow`, `OptionsWindow`, `PresetManagerWindow`, `FuseSelectorWindow`
  (chip bits generated in code-behind, not XAML), `UpdateWindow`,
  `EnterTextDialog`, `MessageDialog`.
- **Services** — `UiConsoleSink` (colored `SelectableTextBlock.Inlines`),
  `UiMsgBoxProvider` (owns modal dialogs), both registered in `App.axaml.cs`.
- **macOS integration** — search `/opt/homebrew/{bin,etc}`,
  `/usr/local/{bin,etc}`, `/etc`, `/opt/local` for `avrdude` and
  `avrdude.conf`; enumerate serial ports from `/dev/cu.usbmodem*`,
  `cu.usbserial*`, `cu.SLAB_*`, `cu.wchusbserial*`, `cu.Bluetooth*`; store
  config under `~/Library/Application Support/AvrdudeUI/` (portable mode via
  `portable.txt` still bypasses).
- **Bundle + icon** — `build/build.sh` publishes with `dotnet publish
  -r osx-arm64 --self-contained`, assembles `.app` around
  `build/AvrdudeUI.app.template/`, and ad-hoc codesigns. Custom `.icns` icon
  generated by `build/icon/generate.py` (PIL master) + `make-icns.sh` (sips
  + iconutil).
- **CI + Release** — `.github/workflows/ci.yml` (build + smoke test on
  Ubuntu / macOS / Windows; `avrdude` installed on Unix runners) and
  `.github/workflows/release.yml` (tag push `[0-9]+.[0-9]+.[0-9]+*` builds
  `.app.tar.gz`, `linux-x64.tar.gz`, `windows-x64.zip` and attaches them to
  a GitHub Release).
- **CoreSmokeTest** — headless harness under `tools/CoreSmokeTest/` that
  exercises Core end-to-end with no UI thread.
- **AGENTS.md** — field guide for AI coding agents (11 non-obvious
  constraints, layout, build/test commands).

### Changed

- Target framework: WinForms / .NET Framework 2.0 → Avalonia 12 / .NET 10.
- `CmdLine` reads from an `AvrdudeSettings` DTO instead of `Form1` directly,
  removing all UI coupling from Core.
- `UpdateCheck` swaps `WebRequest` / `ServicePointManager` (TLS 1.2/1.3
  manual selection) for `HttpClient`.
- `Presets.SetDataSource(ComboBox)` removed; UI binds directly to
  `Presets.BindingSource`.
- `Language.Apply(Control)` removed; XAML views apply translations directly.
- Config-file / preset persistence path is now
  `~/Library/Application Support/AvrdudeUI/` on macOS (`%APPDATA%\AvrdudeUI\`
  on Windows, `$XDG_CONFIG_HOME/AvrdudeUI/` on Linux).

### Fixed

- **macOS 26 startup crash** in `libicucore` (`umutablecptrie_buildImmutable`)
  when .NET built a culture-sensitive search iterator for `String.IndexOf`.
  Resolved with `InvariantGlobalization=true` in the csproj; all
  string comparisons here are ASCII so this is safe.
- **First-run crash** on macOS/Linux because `~/Library/Application
  Support/AvrdudeUI/` didn't exist yet — `XmlFile.Read()` throws
  `DirectoryNotFoundException` (not `FileNotFoundException`) in that
  scenario. Both `Config.Load()` and `Presets.Load()` now catch it.
- **"No window opens" bug** — `UiMsgBoxProvider` was blocking the UI thread
  via `ShowDialog(...).GetAwaiter().GetResult()`. When an error dialog fired
  during startup it self-deadlocked the dispatcher and the main window
  rendered at 0×0. Error/Warning/Info are now fire-and-forget non-modal;
  only `Confirm` blocks, and only when called from a background thread.
- **Cross-thread `SolidColorBrush` allocation** in `UiConsoleSink` triggered
  Avalonia's compositor `VerifyAccess` during render. Brush construction
  and inline appends now happen inside `Dispatcher.UIThread.Post`; brushes
  are cached per RGBA to avoid re-alloc churn.
- **`avr-size is missing` startup error** downgraded to a warning with an
  install hint (`brew tap osx-cross/avr && brew install avr-binutils`),
  since Homebrew's `avrdude` formula doesn't include it and the tool is
  only used for the flash/EEPROM usage readout.

### Removed

- 32-bit Mono support (dead on modern macOS anyway).
- The bundled `avrdude.exe` / `avr-size.exe` — AvrdudeUI relies on the
  system `avrdude` from Homebrew.
- Inno Setup installer scripts — replaced by the `.app` / `.tar.gz` / `.zip`
  artifacts from the release workflow.
- `System.Windows.Forms` references from every ported file.
